The Journey Begins

Just wrapped up the DeepLearning.ai "Building & Evaluating AI Agents" course, and I'm excited to share what I learned by building something real.

Agents feel like the next inflection point: they don't just answer questions, they use tools to get real work done. After the course, I pointed that power at something from my day-job—battery energy storage trading—and the result is BESS-Coding-Agent.

What It Does

The BESS Coding Agent is designed to solve real-world problems in energy trading:

  • Connects to Modo Energy's Day-Ahead-Market API for real-time energy pricing data
  • Uses SmolAgents LLM + custom tools to analyze energy prices and market conditions
  • Provides actionable insights telling storage owners exactly when to charge, when to discharge, and what the cycle profit looks like
  • Chat UI interface for natural language interaction with complex energy data
  • Phoenix dashboard integration streams every tool call and model decision for easy debugging

Technical Implementation

Building this agent involved several key technical decisions:

SmolAgents Framework

I chose SmolAgents for its lightweight approach to building LLM-powered agents. It provided the perfect balance between functionality and simplicity for this project.

Custom Tool Development

Created specialized tools for:

  • API integration with Modo Energy's pricing feeds
  • Battery optimization calculations
  • Profit analysis and reporting
  • Market condition assessment

Observability with Phoenix

Integrated Phoenix tracing to monitor every decision the agent makes. This transparency is crucial when dealing with financial trading decisions.

Why I'm Excited

After seeing this work on a real dataset, I'm re-thinking how AI can slot into every workflow I touch. The ability to have natural conversations about complex energy market data and get actionable trading recommendations feels transformative.

This project demonstrates that agents aren't just fancy chatbots—they're tools that can understand context, use specialized tools, and provide real business value.

Key Learnings

  1. Tool Design Matters: The quality of your custom tools directly impacts agent performance
  2. Observability is Critical: When dealing with financial data, you need to see every decision step
  3. Domain Knowledge Integration: Combining LLM capabilities with industry-specific logic creates powerful solutions
  4. User Experience Focus: A chat interface makes complex data accessible to non-technical users
